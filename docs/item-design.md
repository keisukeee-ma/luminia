# 問題案 具体化 — 課題別アイテム設計

設計ドキュメント（`plans/tingly-jingling-simon.md`）の各課題を、実際に出題できるレベルまで具体化する。

## 出題方式の2分類
- **生成型 (procedural)**: パラメータで無限に項目を生成。難易度＝パラメータで一意に決まるためIRT不要、規準化が容易。
- **バンク型 (curated)**: 人手で項目を作成。項目難易度はIRTで推定する。日本語コンテンツが必要なものはこちら。

各項目の共通フィールド: `item_id, ability, task_id, difficulty(暫定1-5), 取得データ`

# Gs 処理速度（生成型）

## 記号変換（Digit-Symbol Coding）
- **対応表**（9種、課題開始時に上部固定表示）:
  `1=○ 2=△ 3=□ 4=◇ 5=▽ 6=☆ 7=＋ 8=∽ 9=≡`
- **出題**: 記号を1つ提示 → 対応する数字をテンキー入力。これを30秒間ストリームで連続。
- **難易度**: 固定（速度課題なので項目難易度は一定、量で測る）。
- **生成**: 記号を一様ランダム提示（直前と重複しない）。
- **取得データ**: item毎に `symbol, response, correct, rt_ms`。集計で 正答数・中央RT・RT SD・エラー率・前半/後半RT差。

## 一致判定（Pattern comparison）
- **出題**: 2つの文字列（5〜7文字のランダム英数字）を左右提示 → 「同じ/違う」を2ボタンで判定。
  - 例: `ZXKDQ` / `ZXKDQ` → 同じ
  - 例: `ZXKDQ` / `ZXKDO` → 違う（末尾1字）
- **難易度パラメータ**: 文字列長（5/6/7）と相違位置（末尾＝易、中央＝難）。
- **取得データ**: `left, right, is_same, response, correct, rt_ms`。集計で 正答率・中央RT・false alarm率。

# Gsm 作業記憶（生成型）

## 数唱（Digit span forward）
- **出題**: 系列全体を数秒まとめて提示（同時提示）→ 提示順にテンキー入力。
- **固定長**: スパン3〜5を各2試行＝全員6問（早期終了なし／易→難の順）。比較・規準化のため固定。
  - スパン3例: `4 8 2` / スパン4例: `7 2 9 5` / スパン5例: `3 9 1 6 4`
- **生成**: 各桁を一様ランダム（連続同一桁・自明な連続列を避ける）。
- **取得データ**: `span, sequence, response, correct, 系列内正答数(部分点), 入力rt`。集計で 正答数(0〜10)＋ 到達最大スパン。

## 逆唱（Digit span backward）
- 数唱と同じ提示、**逆順**で入力。スパン3例 `5 1 8` → 入力 `8 1 5`。
- 取得データは数唱と同じ（順唱・逆唱を別taskとして保存）。

## 空間スパン（Corsi / グリッド）
- **出題**: 4×4グリッドのマスが順に点灯（各 600ms）→ 同順にタップ。
- **固定長**: 数唱と同様に固定試行（早期終了なし）で実施する方針。
- **生成**: 隣接しすぎない座標列をランダム生成。
- **取得データ**: `span, cell_sequence, tap_sequence, correct, tap_rt`。集計で 正答数＋到達最大スパン（視空間）。

# Gf 流動性推論

## 行列推理（Matrix reasoning）※バンク型（SVG生成）
- **形式**: 3×3行列、右下が欠落、6択から選ぶ。
- **ルール体系**（difficultyの根拠）:
  - 難1: 1属性変化（行ごとに図形の数 1→2→3）
  - 難2: 2属性独立変化（行=数、列=色/塗り）
  - 難3: 回転 + 数の進行
  - 難4: 重ね合わせ（行のA+B=C / XOR）
  - 難5: 複合（回転＋増減＋論理演算）
- **具体例（難2）**: 行＝図形の個数 1,2,3 / 列＝図形種 ○,△,□。欠落＝3個の□。誤答は「2個の□」「3個の△」など属性を1つ間違えたdistractor。
- **取得データ**: `item_id, difficulty, response, correct, rt_ms, distractor_type(数/色/回転の誤り)`。
- **MVP方針**: SVGで属性（種類・数・色・回転・塗り）をパラメータ化し半生成。

## 数列完成（Series completion）※生成型
- **出題**: 数列5項提示 → 次の項を入力 or 4択。
  - 等差: `3 7 11 15 ?` → 19（難1）
  - 等比: `2 4 8 16 ?` → 32（難2）
  - 二階差分: `2 3 5 8 12 ?` → 17（差が1,2,3,4,5）（難3）
  - 平方: `1 4 9 16 25 ?` → 36（難3）
  - フィボナッチ: `1 1 2 3 5 8 ?` → 13（難4）
  - 交互2系列: `1 10 2 20 3 ?` → 30（難4）
- **生成**: 規則タイプ＋係数をパラメータ化。
- **取得データ**: `rule_type, sequence, response, correct, rt_ms`。

## 図形系列 ※バンク型（SVG生成）
- 図形が一定規則で変化（45°ずつ回転 / 辺が1本ずつ増加）→ 続きを4択。

# Glr 長期記憶（バンク型・コンテンツ必須）

## ペア連想学習
- **学習ペア例**（無関連語ペア4組、各2.2秒提示）:
  `海 — 鉛筆 / 山 — 時計 / 川 — 椅子 / 空 — 鍵`
  ※意味的に無関連にするのが要点（既存知識で覚えられないように）。
- **干渉**: 一桁の足し算/引き算3問。
- **再生**: 左語を提示 → 右語を4択（`海 — ? : 鉛筆/消しゴム/定規/ペン`）。
- **取得データ**: `pair_id, cue, response, correct, rt_ms`。集計で 正再生数。

## 単語リスト学習（自由再生 + 遅延再生）
- **リスト例（15語、3カテゴリ×5でクラスタ分析可能）**:
  - 動物: いぬ・ねこ・うま・くま・とり
  - 果物: りんご・みかん・ぶどう・いちご・もも
  - 道具: はさみ・かなづち・ものさし・ドライバー・のこぎり
  - 提示は**カテゴリ混在のランダム順**で各1.5秒。
- **即時再生**: 自由テキスト入力（覚えた語を列挙）。
- **遅延再生**: セッション終盤に再度（手がかりなし）。
- **取得データ**: `recalled_words[], 正再生数, 侵入語(intrusions), 系列位置(初頭/新近), 意味クラスタ数, 遅延正再生数`。

## 言語流暢性（任意・Gr）
- **音韻**: 「か」で始まる語を60秒で列挙。
- **意味**: 「動物」の名前を60秒で列挙。
- **取得データ**: `産出語数, 重複数, 規則違反数, クラスタ数, スイッチ数, 時間内産出推移(15秒区間別)`。

# Gv 視覚処理

## 心的回転（Mental rotation）※生成型（SVG）
- **出題**: 基準図形（非対称）と回転候補を提示 → 「同一 / 鏡像」を判定。
  - 図形: 非対称ポリオミノ or 「R」「F」等の非対称文字。
  - 回転角: 0/45/90/135/180°。
- **難易度パラメータ**: 回転角（大きいほど難）+ 鏡像か否か。
- **取得データ**: `angle, is_mirror, response, correct, rt_ms`。集計で **角度×RTの傾き＝心的回転速度**、鏡像弁別エラー率。

## 紙折り（Paper folding）※バンク型（SVG）
- **出題**: 正方形を1〜2回折る → 穴を開ける → 「広げた図」を4択。
- **難易度**: 折り回数（1回＝易、2回＝難）、穴の数。
- **具体例（難1）**: 縦半分に折る → 中央に1穴 → 広げると左右対称に2穴。
- **取得データ**: `fold_count, hole_count, response, correct, rt_ms`。

## 図形閉合（任意）
- 断片化/ノイズ画像 → 何かを4択で同定。閉合速度(CS)。

# Gc 結晶性知能（バンク型・対照アンカー）

## 語彙（同義語選択）
- **形式**: 語を提示 → 最も意味が近いものを4択。
  - `克明` → ①詳しく丁寧(正) ②大胆 ③簡潔 ④曖昧（難2）
  - `普遍` → ①すべてに共通して当てはまる(正) ②めずらしい ③一時的 ④個別の（難2）
  - `示唆` → ①それとなく示す(正) ②強く命じる ③完全に否定する ④繰り返す（難3）
  - `脆弱` → ①もろくて弱い(正) ②頑丈 ③巨大 ④高価（難1）
- **取得データ**: `item_id, difficulty, response, correct, rt_ms(参考)`。

## 一般知識（事実4択）
- `日本で最も長い川は?` → 信濃川（②）。他: 利根川/石狩川/北上川（難2）
- `光が1秒で進む距離に最も近いのは?` → 約30万km（難3）
- `「源氏物語」の作者は?` → 紫式部（難1）
- **取得データ**: 語彙と同じ。

## ことわざ・慣用句（日本語固有）
- **形式A（穴埋め4択）**: `猿も木から___` → 落ちる/転ぶ/降りる/滑る（難1）
- **形式B（意味選択）**: `「二階から目薬」の意味は?` → ①もどかしく効果がない(正) ②高い所が好き ③準備が良い ④幸運が続く（難2）
- 他シード: 石の上にも三年 / 急がば回れ / 帯に短したすきに長し / 灯台下暗し
- **取得データ**: 語彙と同じ。

# 項目バンクの最小数量（MVP目安）
| 課題 | 方式 | 初期項目数の目安 |
|---|---|---|
| 記号変換・一致判定・数唱・逆唱・空間スパン・数列 | 生成型 | ルール実装のみ（項目バンク不要） |
| 行列推理 | 半生成 | ルール5種×各3＝15 |
| 語彙 | バンク | 30（難易度分散） |
| 一般知識 | バンク | 30 |
| ことわざ | バンク | 30 |
| ペア連想 | バンク | 無関連語ペア 20組 |
| 単語リスト | バンク | カテゴリ語 30語 |
| 心的回転・紙折り | 半生成/バンク | 回転=ルール / 紙折り=12 |

> 生成型を優先実装すると、コンテンツ作成コストを抑えつつ6能力中4〜5能力をカバーできる。バンク型（語彙/知識/ことわざ）は別途項目を蓄積する。
