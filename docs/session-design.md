# セッション設計 — 10分・低離脱

## 目標
- **所要時間 約10分（計測部 ≈ 9〜10分）**で6能力すべてを測る。
- **途中離脱を最小化**する構成・演出。

## 設計上のトレードオフ
「正確に測る＝項目を増やす＝長くなる＝離脱する」。10分制約を守るため、
**各能力 原則1課題（処理速度・作業記憶のみ2課題）**に絞り、残りは拡張モード（任意）へ回す。
信頼性は規準データ蓄積で補う。

## 順序の設計意図（離脱対策）

1. **簡単な勝ちで始める** — 記号変換は誰でもできてゲーム的。最初の成功体験で離脱率↓。
2. **記憶の符号化を冒頭、再生を末尾** — 間の他課題が自然な「遅延・干渉」になり、退屈な空白を作らずに遅延再生が成立する。
3. **認知デマンドを交互に** — 速度→記憶→推論→空間→記憶→知識…と種類を変え、単調さ（モノトニー）による飽きを防ぐ。同種を連続させない。
4. **難所は中盤** — 最も負荷の高い行列推理は2〜4番目。疲労前かつ開始直後の警戒も解けたタイミング。
5. **終わりを良い体験に** — 末尾は単語再生（達成感）→即「脳年齢」結果表示で報酬。

## 離脱を防ぐUI/演出の原則

- **総量を最初に明示**: 開始画面で「約10分・全11問」。全体進捗バー（ステップ進捗）を常時表示。
- **時間の予測可能性**: 全課題に制限時間、残量はゲージで可視化（数字より不安が少ない）。
- **適応打ち切り**: 数唱・逆唱は2連続失敗で即終了 → 解けない問題を延々続けさせない（最大のフラストレーション源を断つ）。
- **失敗を罰しない**: ○/×は短時間、即時スコア数値は見せない（不安を煽らない）。トーンは励まし基調。
- **準備画面は短く**: 説明は1〜2行＋例示1つ。長文説明自体が離脱点。
- **部分保存**: 途中離脱でも計測済みステップは保存（既存方針）。

## 属性入力の分割（重要な離脱点）

7属性（年齢/性別/郵便番号/職業/職種/業種/役職）を**計測前に全部聞くと離脱する**。

- **計測前**: 規準化に必須の **年齢帯・性別のみ**（2項目）。
- **結果表示後**: 残り（郵便番号・職業・職種・業種・役職）を
  「より正確な比較のために」と任意入力。
  → 結果（脳年齢）を見た後はモチベーションが高く、入力されやすい。

## 時間が押した場合の削減順（優先度・低い順）
1. ことわざ 2問（Gc は他2課題で代替可）
2. 数列完成 3問（Gf は行列で代替可）
3. 一般知識 2問
→ これで約8分まで短縮できる。逆に Gs・Gsm・Glr・主要Gf は削らない（脳年齢の主軸）。
